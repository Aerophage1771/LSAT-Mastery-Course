# RC Concept Depth Parity

**Purpose:** Define the roadmap bet for addressing the depth disparity between Reading Comprehension (RC) and Logical Reasoning (LR) core modules.
**Audience:** Maintainers, content editors, and future agents planning course improvements.
**Status:** draft
**Source of truth:** yes
**Last reviewed:** 2026-04-20
**Related docs:** [README.md](./README.md), [vision-and-decision-filter.md](./vision-and-decision-filter.md), [../course-content-map.md](../course-content-map.md)

## Problem

- Imbalanced lesson coverage: The core RC modules (e.g., Inference, Application, Weaken-Strengthen) are structurally "thin" compared to their LR counterparts.
- While LR modules regularly feature "Pattern Recognition" and "Trap Answer Types" lessons, RC core modules often lack these deeper, more analytical lessons, leaving a gap in methodology for students.

## Proposed Direction

- Expand the core RC concept modules to include "Pattern Recognition" and "Trap Answer Types" lessons, achieving structural parity with LR.
- Leverage existing RC passage data to build out repeatable reading patterns and common trap distractors specific to RC question types.

## Why Now

- The course shell and lesson routing are stable. We have a clear structural map ([course-content-map.md](../course-content-map.md)) that highlights exactly where the RC coverage drops off.
- Fixing this disparity directly addresses student confidence and the depth of the product offering.

## Likely Upside

- Creates a stronger, more coherent curriculum.
- Better equips students to handle RC questions by providing the same level of analytical tools they receive for LR.
- Makes the entire course more reusable as a system for generating future lessons.

## First Practical Milestone

- Draft the "Pattern Recognition" and "Trap Answer Types" lessons for a high-priority RC module (e.g., Module 24: Main Idea or Module 25: Inference).
- Ensure the new lessons pass standard validation checks and integrate correctly into the course catalog map.
