# Module 50: RC Quick Reference Guide -- Website Content Improvement Report

**Scope:** The 1 website lesson (50-1) evaluated as a standalone reference document.
**Date:** 2026-04-01

---

## Executive Summary

Module 50 is a comprehensive RC question-type taxonomy and strategy guide. It covers 14 question types organized into 4 groups, with each type receiving a consistent breakdown (Core Task, Approach, Trap, Stem Identifiers). This is the definitive reference for the RC section and is well-structured overall. The primary improvement opportunities are:

1. **No passage-reading strategy section.** The guide covers question types exhaustively but provides zero guidance on how to read the passage itself -- arguably the most important RC skill. There is no advice on passage structure identification, paragraph-by-paragraph summarization, active reading techniques, or time management for the reading phase.
2. **No comparative passage strategy.** Given that 2 of the 4 practice passages (M46, M49) are comparative, the guide should include specific guidance for handling paired passages.
3. **The Quick Reference Guide at the bottom is excellent** -- it distills each question type into a one-line goal and one trap. This is the most valuable standalone component.
4. **No worked examples or PT references.** The guide is entirely abstract. Even one short worked example per group (or per question type) would dramatically improve its usability.
5. **No cross-reference to practice modules.** Students cannot connect the taxonomy to the specific practice passages where each question type appears.

## Section-by-Section Review

### Group 1: The Macro Questions (4 types)

**Types covered:** Main Idea, Primary Purpose, Structure/Organization, Tone/Author Attitude

**Strengths:**
- Each type has a clear "Core Task" definition.
- The "Approach" sections provide step-by-step procedures.
- The "Trap" callouts identify the #1 distractor pattern for each type.
- The "Stem Identifiers" sections list the exact question stems students will see.
- Main Idea approach is strong: "Paragraph Synthesis -> Build Blueprint -> Predict & Match" with a clear backup plan.

**Issues:**
- **Primary Purpose lacks a verb list.** The approach says "Select the Precise Verb" but does not provide a list of common purpose verbs (e.g., defend, critique, explain, propose, evaluate, compare). This is the most commonly needed reference for this question type.
- **Tone/Author Attitude approach is good but thin.** It covers identifying "charged language" but does not include a tone vocabulary list (e.g., skeptical, cautiously optimistic, dismissive, approving).
- **No distinction between Main Idea and Primary Purpose.** These are the two most commonly confused question types. A brief comparison note would be valuable.

**Suggested edits:**
- Add a common purpose verbs list to the Primary Purpose section.
- Add a tone vocabulary spectrum to the Tone/Author Attitude section.
- Add a brief comparison note distinguishing Main Idea from Primary Purpose.

---

### Group 2: The Role & Function Questions (3 types)

**Types covered:** Statement Function, Paragraph Function, Phrase Meaning

**Strengths:**
- Statement Function approach is clean: "Isolate Local Argument -> Define Role -> Predict and Match."
- Paragraph Function includes position-based guidance (first/middle/final) -- practical and actionable.
- Phrase Meaning includes the "Substitution Test" -- the single most effective technique for this question type.
- The "Content Dump" trap for Statement Function is correctly identified.

**Issues:**
- **No distinction between Statement Function and Paragraph Function approach.** The guide treats them as separate types but the approaches are very similar. A brief note on scale (detail vs. whole paragraph) would help students distinguish them.
- **Phrase Meaning could include a note about metaphorical/ironic usage.** The "Standard Definition" trap is correctly identified, but the approach does not explicitly tell students to consider non-literal usage.

**Suggested edits:**
- Add a brief note distinguishing Statement Function (detail-level) from Paragraph Function (paragraph-level).
- Add a note about metaphorical/ironic usage to the Phrase Meaning section.

---

### Group 3: The Inference Questions (3 types)

**Types covered:** Standard Inference, Author Viewpoint, 3rd-Party Viewpoint

**Strengths:**
- Standard Inference approach correctly emphasizes the "skeptical mindset" and textual evidence requirement.
- The distinction between Author Viewpoint and 3rd-Party Viewpoint is well-drawn.
- The "Misattribution" trap for Author Viewpoint is correctly identified -- this is a critical error pattern.
- The "Mix-Up" trap for 3rd-Party Viewpoint is the mirror image, creating a coherent pair.

**Issues:**
- **No guidance on "degree of support" for inferences.** The guide says to find the answer requiring the "smallest possible logical leap" but does not provide examples of what a small vs. large leap looks like.
- **No mention of "EXCEPT" or "LEAST" variants.** Several practice questions (e.g., Q12 in Module 47) use EXCEPT format. The guide should note that these invert the task.

**Suggested edits:**
- Add a brief note on EXCEPT/LEAST question variants and how they change the task.
- Consider adding a brief example of what a "small logical leap" looks like vs. an "out of scope" inference.

---

### Group 4: The "LR in RC" Questions (4 types)

**Types covered:** Analogy/Parallel Reasoning, Application/Hypothetical, Passage Principle, Weaken/Strengthen

**Strengths:**
- Analogy approach is strong: "Isolate Scenario -> Create Abstract Rule -> Test Choices."
- Application/Hypothetical correctly uses a "Checklist" approach -- practical and systematic.
- Weaken/Strengthen correctly maps to the standard LR framework (Evidence -> Assumption -> Conclusion).
- Passage Principle approach is clean and correctly identifies the generalization task.

**Issues:**
- **No passage completion question type.** Q21 in Module 48 is a passage completion question that does not fit neatly into any of the 14 types. A brief note covering this rare type would make the guide truly exhaustive.
- **No cross-passage question types.** Several questions in Modules 46 and 49 require cross-passage application (using one passage's framework to analyze the other). This is not covered anywhere in the guide.

**Suggested edits:**
- Add a brief note on passage completion questions (rare, functions like main point).
- Add a section or note on cross-passage question variants for comparative passages.

---

### Quick Reference Guide (Bottom Section)

**Strengths:**
- Excellent one-line summaries with goal and #1 trap for each type.
- This is the most valuable standalone component for quick review before a practice section.
- Clean formatting with breakdown component.

**Issues:**
- **Missing question types from the main guide.** The quick reference covers 9 types but the main guide covers 14. The 5 missing from the quick reference are: Structure/Organization, Phrase Meaning, 3rd-Party Viewpoint, Application/Hypothetical, and Passage Principle.

**Suggested edits:**
- Add the 5 missing types to the quick reference guide for completeness.

## Structural Issues

### 1. No Passage-Reading Strategy

The guide provides excellent question-answering strategy but zero passage-reading strategy. This is the largest gap. Students need guidance on:
- How to read the passage on first pass (active reading, paragraph summarization)
- Time allocation (how long to spend reading vs. answering)
- What to track while reading (thesis, structure, tone, key terms)
- How to handle different passage types (science, law, humanities, social science)

**Recommendation:** Add a "Group 0: Passage Reading Strategy" section before Group 1 covering active reading techniques, time management, and passage-type awareness.

### 2. No Comparative Passage Guidance

The guide does not mention comparative passages at all. Given that they represent 25% of a standard RC section, this is a significant gap.

**Recommendation:** Add a section on comparative passage strategy covering: reading order, relationship identification (agree/disagree/extend/apply), and cross-passage question types.

### 3. Quick Reference Guide Is Incomplete

The quick reference at the bottom covers only 9 of the 14 question types defined in the main guide. This inconsistency could confuse students who use the quick reference as their primary tool.

**Recommendation:** Add the 5 missing types to the quick reference.

### 4. No Worked Examples

The guide is entirely abstract. A brief worked example (even 2-3 sentences) for each question type would make the approach sections much more concrete.

**Recommendation:** Add a short "Example" subsection to each question type showing the approach in action. These could reference specific questions from the practice modules.
