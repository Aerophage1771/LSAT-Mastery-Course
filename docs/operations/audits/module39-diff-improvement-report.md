# Module 39: Structure -- Website Content Improvement Report

**Scope:** The 2 website lessons (39-1 through 39-2) evaluated as a standalone course unit.
**Date:** 2026-04-01

---

## Executive Summary

Module 39 covers Structure questions -- a question type that asks students to map the logical organization of an entire passage. At 48 appearances, this is a moderate-frequency type that tests advanced comprehension. The two lessons are well-organized and the "blueprint" concept is the module's central metaphor. Key findings:

1. **Zero practice questions** -- neither lesson includes a question-card or complete worked example.
2. **The "blueprint" concept is the module's strongest element** -- both lessons consistently frame Structure questions as requiring students to build a logical map of the passage, and the example blueprint in 39-2 is one of the best teaching tools across all RC modules.
3. **Strong relationship to Paragraph Function (M36)** -- Structure questions are essentially Paragraph Function scaled to the whole passage. The module does not explicitly acknowledge this connection.
4. **Three stem categories with a unique "Relational Structure" category** -- the stem breakdown includes a category for questions about the relationship between specific paragraphs, which bridges Structure and Paragraph Function questions.
5. **Good transition-word guidance** -- 39-2's tie-breaker about paragraph transitions ("However," "For example," "Consequently") is practical and actionable.

## Lesson-by-Lesson Review

### Lesson 39-1: Germaine's Guide to RC Structure Questions (~14 blocks)

**Role:** Module introduction, concept overview, and stem identification guide.

**Strengths:**
- Clear opening that correctly frames the task as creating a "blueprint" of the passage's organization
- The four-item organizational pattern list (challenge a view, evaluate solutions, trace history, support + address counterargument) covers the most common passage structures
- Three stem categories (Direct Inquiry, Sequential Flow, Relational Structure) are well-differentiated
- The Relational Structure category is valuable -- it identifies questions that target specific paragraph relationships, bridging Structure and Paragraph Function
- The Overarching Takeaway correctly emphasizes abstract functional language over topic summaries
- Frequency data (48 appearances) and the "overall comprehension" framing establish appropriate importance

**Issues:**
- **No practice question.** The organizational patterns are listed but never tested.
- **No examples of the four organizational patterns.** Each pattern is described in one sentence but not illustrated with a passage outline.
- **No connection to Paragraph Function (M36).** Structure questions are the natural extension of paragraph-level analysis, and an explicit link would help students see the progression.
- **The organizational patterns list is too short.** Common structures like "compare and contrast two theories," "trace the evolution of a concept," and "present a paradox and resolve it" are missing.

**Suggested edits:**
- Expand the organizational patterns list to 6-8 items, covering the full range of common passage structures.
- Add brief passage-outline examples for each pattern.
- Add a cross-reference to Paragraph Function (M36): "Structure questions build directly on paragraph-level analysis. If you can identify each paragraph's function (Module 36), you can construct the passage's structure."
- Add a stem identification exercise.

---

### Lesson 39-2: Step-by-Step Approach (~13 blocks)

**Role:** The complete method for solving Structure questions.

**Strengths:**
- Lean two-step method: Create a Passage Blueprint, Match to Answer Choices
- The example blueprint ("Paragraph 1: Introduces a common theory. Paragraph 2: Presents evidence that challenges this theory...") is one of the best teaching tools in the RC modules -- concrete, functional, and immediately replicable
- Step 2's elimination criterion ("If an answer choice misrepresents the function or sequence of even one part of the passage, it is incorrect") is precise and actionable
- Tie-Breaker #1 (Paragraph Transition Test) using signpost words is practical and grounded in the text
- Tie-Breaker #2 (Focus on the Verbs) teaches precision in abstract descriptions ("refuting" vs. "qualifying")

**Issues:**
- **No question-card.** The method is demonstrated but never practiced.
- **The blueprint method is only a two-step process.** Compared to three-step methods in other modules, this feels lean. The blueprint creation step is doing heavy lifting and could be broken into sub-steps (e.g., read, annotate each paragraph, then assemble).
- **No worked example with answer choices.** The blueprint example shows how to create the map but does not show how to evaluate answer choices against it.
- **The verb-based tie-breaker appears in both M36 and M39.** This overlap is minor but could be acknowledged with a cross-reference.

**Suggested edits:**
- Break Step 1 into sub-steps: (1a) Read and annotate each paragraph's function, (1b) Identify transitions between paragraphs, (1c) Assemble the full blueprint.
- Add a worked example showing 4-5 answer choices evaluated against the example blueprint.
- Add a question-card after the method section.
- Add a cross-reference to M36's verb-based approach.

---

## Structural Issues

### 1. No Practice Questions

Both lessons are pure instruction. Structure questions require students to synthesize the entire passage, making practice particularly important.

**Recommendation:** Add at least one question-card to 39-2.

### 2. Missing Cross-Module Connections

Structure questions build directly on Paragraph Function skills (M36) and are closely related to Passage Function (M37). Neither connection is acknowledged.

**Recommendation:** Add cross-references to both M36 and M37 in 39-1.

### 3. Organizational Patterns List Is Too Short

Four patterns do not cover the full range of LSAT passage structures. Common patterns like compare-and-contrast and paradox-resolution are missing.

**Recommendation:** Expand to 6-8 patterns with brief examples.
