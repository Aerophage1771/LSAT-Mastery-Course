# Module 4: Parallel Reasoning -- Website Content Improvement Report

**Scope:** The 12 website lessons (4-1 through 4-12) evaluated as a standalone course unit.
**Date:** 2026-04-01

---

## Executive Summary

Module 4 is a well-structured and comprehensive unit that covers Parallel Reasoning from introduction through advanced applications. The 12 lessons progress logically from abstraction basics to specialized patterns (conditional chains, eliminative, principle application, quantifier) to advanced material (statistical rebuttal, correlational rule). The primary improvement opportunities are:

1. **No real LSAT question-cards in any lesson** -- unlike Modules 1-3 where advanced lessons each had a PT question, every worked example in Module 4 uses illustrative/invented stimuli. This is the module's single biggest gap.
2. **Lesson 4-3 is too long at 98 blocks** -- it covers all 9 argument structures in a single lesson. This is a reference-catalog lesson that tries to serve as both instructional content and cheat sheet.
3. **Lesson 4-1 has significant overlap with the website diff version** -- the diff alternate in the Mastery Course carries over substantial content from the website (Abstraction Process, What to Preserve/Discard, Common Structures Preview, 4-Step Method Preview) that is then re-taught in 4-2 and 4-3. The introduction should be leaner.
4. **Worked examples lack "try it yourself" prompts** -- every worked example immediately follows with the solution. No lesson pauses to let the student attempt the question before reading the analysis.
5. **The Reference Guide (4-12) is solid but missing frequency data** -- unlike the 4-3 frequency table, the reference guide omits pattern frequency, "unless" translation rules, and the diagramming method from 4-9.

## Lesson-by-Lesson Review

### Lesson 4-1: MODULE 4: Parallel Reasoning (49 blocks)

**Role:** Module introduction and overview.

**Strengths:**
- Clear Question Goal statement opens the lesson with precision
- The "Why This Matters" section effectively positions Parallel Reasoning as the culmination of Modules 1-3
- The Frequency & Timing callout (1-2 per section, 2-2.5 minutes) is practical and actionable
- The Parallel Reasoning vs. Parallel Flaw callout is critical -- students confuse these constantly
- The breakdown showing Biology vs. Workplace examples with identical abstract patterns is the best teaching moment in the lesson
- The question stem recognition table with 5 stem variants is useful reference material

**Issues:**
- **Scope creep in the diff version.** The website content for 4-1 is 49 blocks, but the Mastery Course diff alternate expands to include the full Abstraction Process section (blockquote example, variable mapping table, What to Preserve/Discard table), the Common Logical Structures Preview table (9 patterns), and the 4-Step Method Preview. All of this is re-taught more thoroughly in 4-2 and 4-3.
- **No practice question.** The lesson is entirely theoretical. Even a single "identify the abstract pattern of this 3-sentence argument" exercise would improve retention.
- **The "Your First Parallel Reasoning Question" section (blocks 41-46) is weak.** It describes a walkthrough process but doesn't present a real question with answer choices. The student reads a process description without actually doing anything.
- **The question stem table (block 15) lists 5 stems** but doesn't clarify that "illustrates the principle" stems may require different handling than "similar in its reasoning" stems -- this nuance is only covered in the Principle Application lesson (4-6).

**Suggested edits:**
- Trim the diff alternate to remove the Abstraction Process and Common Structures Preview sections. Replace with a forward reference to 4-2 and 4-3.
- Replace the walkthrough description with a genuine mini-exercise: present 3 short arguments, ask the student to match two that share a logical skeleton.
- Add a note in the question stem table flagging the "principle" stem as requiring different handling.

### Lesson 4-2: The 4-Step Method (51 blocks)

**Role:** The complete solving method for Parallel Reasoning questions.

**Strengths:**
- Excellent 4-step structure: Identify -> Abstract -> Eliminate -> Verify
- The timing breakdown (30s stimulus, 15s per answer first pass, 30s confirmation) is uniquely practical
- The Abstraction Table (block 20) mapping 9 concrete-to-abstract translations is one of the strongest tools in the module
- The Falcons argument (Affirming the Consequent) is a perfect choice -- it shows an invalid argument that students must recognize as invalid
- The Conclusion-First Strategy callout is the single most important time-saver in the module
- The two-pass elimination approach (Quick Scan then Structural Check) gives a realistic workflow

**Issues:**
- **No real LSAT question.** The Full Worked Example (blocks 41-48) uses the historic landmark Modus Ponens argument, which is illustrative but not an actual LSAT stimulus. There are no answer choices to evaluate.
- **The worked example is too clean.** It uses a straightforward Modus Ponens, which is the easiest pattern. A harder example (conditional chain or eliminative with a conditional conclusion) would better demonstrate the method's value.
- **Step 3 advice is generic.** The "Match by Elimination" section tells students to "scan the answer choices" but doesn't show what this looks like with actual answer choices to evaluate.
- **The Structural Check table (block 33) is excellent** but appears only in the diff version -- the primary content omits the "When To Diagram" guidance and the "Track Record -> Future Probably" illustrative pattern callout.

**Suggested edits:**
- Add a real LSAT question-card (PT question) so students can practice the method with actual answer choices before seeing the analysis.
- Replace or supplement the historic landmark example with a harder pattern (eliminative or conditional chain).
- Add a "try this yourself first" prompt before the worked example analysis.

### Lesson 4-3: Common Argument Structures (98 blocks)

**Role:** Master reference for all 9 recurring argument patterns.

**Strengths:**
- The frequency table (block 4) with exact percentages is excellent -- gives students a clear prioritization framework
- Covers all 9 patterns comprehensively: Conditional Chain, Modus Ponens/Tollens, Eliminative, Principle Application, Quantifier, Causal Explanation, Analogy, Ad Hominem/Tu Quoque, Inductive Prediction
- Each pattern gets the same treatment: Abstract Pattern table, blockquote example, indicator words, common variations
- The Quantifier Strength Hierarchy table (block 57) is one of the most important reference tools in the entire module
- The Pattern Identification Process (block 96) with a decision-tree process block is actionable
- The callouts about invalid twins (Affirming Consequent, Denying Antecedent) and the key structural features for each pattern are well-placed

**Issues:**
- **98 blocks is far too long for a single lesson.** This is effectively 9 mini-lessons packed into one. Students will lose focus well before reaching the Analogy pattern (block 70).
- **No practice questions.** The lesson introduces 9 distinct argument patterns without a single exercise. Even one question per major pattern (conditional chain, eliminative, principle) would dramatically improve learning.
- **The later patterns get less attention.** Patterns 7-9 (Analogy, Ad Hominem, Inductive Prediction) each get only 6-8 blocks compared to 12-15 for the top patterns. Given their low frequency (~2-5%), this may be appropriate, but it makes the lesson feel front-heavy.
- **Overlap with individual deep-dive lessons.** Lesson 4-3 covers conditional chains, which are then re-covered in 4-4. The eliminative pattern is re-covered in 4-5. Principle application is re-covered in 4-6. Quantifier arguments are re-covered in 4-7. This means students read about each major pattern twice -- once here and once in its dedicated lesson.

**Suggested edits:**
- Consider splitting into two lessons: "Core Patterns" (conditional chain, Modus Ponens/Tollens, eliminative, principle application) and "Additional Patterns" (quantifier, causal, analogy, ad hominem, inductive prediction).
- Add at least one mini-exercise for the top 3 patterns: give a short argument, ask the student to name the pattern.
- Reduce overlap with the individual deep-dive lessons by making 4-3 a concise preview and letting the deep-dives own the detailed treatment. Or: make 4-3 the definitive reference and trim the individual lessons' introductory material.

### Lesson 4-4: The Conditional Chain (34 blocks)

**Role:** Deep dive into the most common Parallel Reasoning pattern.

**Strengths:**
- Tight and focused at 34 blocks covering one pattern thoroughly
- The conditional translation table (block 14) showing 5 natural language phrasings and their symbolic forms is excellent
- The "Only" Trap callout correctly identifies one of the most common student errors in conditional logic
- The Variations table (block 19) distinguishing Basic Chain, Extended Chain, Chain + Application, and Contrapositive Chain is comprehensive
- The Worked Example with full 4-step method walkthrough and 4-item wrong-answer analysis is well-structured
- The Common Traps table (block 31) naming Modus Tollens Disguise, Broken Chain, Disjunction Swap, and the Application Trap is actionable

**Issues:**
- **No real LSAT question-card.** The worked example uses an illustrative stimulus (spicy meals/heartburn/grouchy). A real PT question would validate the method against actual test difficulty.
- **Overlap with 4-3.** The Conditional Chain is already covered in blocks 6-15 of Lesson 4-3. Lesson 4-4 goes deeper (variations, translation table, "Only" trap), which is valuable, but the opening material repeats 4-3.
- **The wrong-answer analysis (block 27) uses descriptive letters** (A, B, C, D) rather than actual answer choice text. This is less engaging than presenting full answer choice text.

**Suggested edits:**
- Add a real LSAT question-card.
- Open with a direct reference to 4-3's preview rather than re-defining the conditional chain pattern.
- Add a "try it yourself" prompt before the worked example solution.

### Lesson 4-5: The Eliminative Argument (42 blocks)

**Role:** Deep dive into the eliminative/disjunctive syllogism pattern.

**Strengths:**
- Excellent recognition table (block 13) showing 5 ways the LSAT presents disjunctions without using "either/or" explicitly
- The separate elimination phrasings table (block 15) is uniquely helpful -- it catalogs the ways the LSAT eliminates options
- The 3 variations (Binary, Multi-Option, Conditional) with full tables and blockquote examples are thorough
- The "Conditional Framing Is Critical" callout addresses a common mistake: mixing definite and conditional conclusions
- The Wrong-Answer Analysis (block 35) with detailed explanations for each wrong answer type is well-structured
- The Common Traps table (block 39) identifying Positive Selection, The Dilemma, and The Abandonment is valuable

**Issues:**
- **No real LSAT question-card.** Same gap as 4-4.
- **The worked example stimulus (Carl's Coffee)** is good but reappears across multiple examples in the lesson. Using a second, distinct stimulus would show the pattern's range.
- **The "Active vs. Passive Elimination" distinction** (positive selection vs. negative elimination) is one of the most important teaching moments but appears only in the trap table and wrong-answer analysis. It deserves its own callout or breakdown.

**Suggested edits:**
- Add a real LSAT question-card.
- Promote the "Active vs. Passive Elimination" concept to a standalone callout with side-by-side examples.
- Add a second worked example with a multi-option elimination.

## Structural Issues

### 1. No Real LSAT Question-Cards Anywhere in the Module

This is Module 4's single biggest gap. Modules 1-3's advanced lessons each included real PT question-cards with full worked analyses. Module 4 uses only illustrative/invented stimuli for every worked example and practice question across all 12 lessons. While the illustrative examples are well-chosen and clearly demonstrate each pattern, they lack the complexity, ambiguity, and trap sophistication of real LSAT questions.

**Recommendation:** Add real PT question-cards to at least 4 lessons: one in the method lesson (4-2), one in a core pattern lesson (4-4 or 4-5), one in an advanced lesson (4-9 or 4-10), and one more anywhere. This brings Module 4 in line with the standard set by Modules 1-3.

### 2. Lesson 4-3 Is Too Long and Overlaps Deep-Dive Lessons

At 98 blocks, Lesson 4-3 is the longest in the module and nearly double the length of any other lesson. It covers all 9 patterns that are then individually re-taught in lessons 4-4 through 4-7. Students read about conditional chains twice (4-3 and 4-4), eliminative arguments twice (4-3 and 4-5), etc.

**Recommendation:** Either (a) make 4-3 a concise preview (one table row per pattern + the frequency table + the decision process) and let the individual lessons own the detailed treatment, or (b) split 4-3 into two lessons: Core Patterns and Additional Patterns.

### 3. No "Try It Yourself" Prompts Before Any Worked Example

Every worked example in the module presents the stimulus and immediately follows with the solution. There is never a pause asking the student to attempt the question before reading the analysis. This is a missed opportunity for active learning.

**Recommendation:** Add a brief "attempt this question before reading the analysis below" callout before every solution section. This is a small change with outsized pedagogical impact.

### 4. Diff Alternates Mirror Website Content Almost Exactly

The Mastery Course diff alternates for lessons 4-1, 4-2, 4-3, and 4-12 are nearly identical to the website content (with minor additions). This means the diff view doesn't add meaningful improvement annotations. The diff alternates should use diff markers to flag specific improvement opportunities.

**Recommendation:** Annotate the diff alternates with `{{+add+}}`, `{{-remove-}}`, `{{~change~}}`, and `{{!note!}}` markers highlighting the highest-impact improvements identified in this report.
