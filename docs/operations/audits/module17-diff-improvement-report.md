# Module 17: Must Be True — Website Content Improvement Report

**Scope:** The 11 website lessons (17-1 through 17-11) evaluated as a standalone course unit.
**Date:** 2026-04-01

---

## Executive Summary

Module 17 is a technically rigorous and well-organized unit. The six reasoning-pattern drills (17-3 through 17-9) each follow a clean concept-intro → worked question-card → full analysis → trap taxonomy structure that is consistent and pedagogically sound. The reference guide (17-11) is genuinely complete, covering the 4-step method, all reasoning types, the certainty test, a quantifier reference, nine common traps, and a test-day checklist — this is the strongest reference guide in the LR module set.

The primary improvement opportunities are:

1. **No "try it first" prompts in any drill lesson.** Every question-card is followed immediately by a full analysis. A single sentence before each analysis would convert passive reading into active practice. This is the highest-priority change across the module.
2. **Lesson 17-6 (High-Difficulty Analysis) has a broken question-card.** The lesson discusses a worked example (Rocktown school board / Dorsey) but does not show the question stem, and the full analysis block references answer choices that are never presented to the student as a proper question-card. This is the most significant structural gap.
3. **Lesson 17-1 omits two reasoning patterns** (Governed Sets & Domains; Policy/Rule/Exception/Purpose/Effect) that appear in the mastery course version and are central to the advanced lessons (17-12, 17-13). The website introduction understates the module's scope.
4. **Lesson 17-2 is tighter than the mastery course version** but omits the strategic distinction between prephrase-first and answer-driven POE paths — the most practical guidance for handling hard questions.
5. **Lesson 17-11 (Reference Guide) is excellent** with minor labeling issues: the summary callout says "Module 17" but Step 2 of the 4-step method says "Factual Components" rather than "Governed Components" (the mastery course upgrade), and the reasoning-types table lists only 6 types while the module covers 8.

## Lesson-by-Lesson Review

### Lesson 17-1: MODULE 17: Must Be True (11 blocks)

**Role:** Module introduction and concept overview.

**Strengths:**
- Clear Question Goal statement at the top.
- The MBT vs. MSS breakdown table is well-structured and addresses the most common source of student confusion upfront.
- The Certainty Standard section and the mental test ("Can I construct ANY scenario...") are precise and actionable.
- The Concept Snapshot (robotics club / computer science / lab fee) is simple, relatable, and correctly analyzed.
- The Key Principle callout is an effective closer.

**Issues:**
- **Two reasoning patterns missing.** The "Common MBT Reasoning Patterns" list has 5 items. The mastery course version has 7, adding "Governed Sets & Domains" and "Policy/Rule/Exception/Purpose/Effect" — both of which are the subjects of entire lessons (17-12, 17-13). A student reading this introduction would not know those patterns exist.
- **No practice question.** The lesson is 11 blocks of pure theory. An introductory lesson can be theory-heavy, but even a mini-exercise after the Concept Snapshot would improve retention.
- **The Certainty Standard section has only one paragraph** (the mental test). The mastery course adds a second paragraph explaining that the proof standard stays fixed whether you prephrase first or use answer-driven POE — this is important framing for the method lesson that follows.
- **No "try it first" prompt** — not applicable here since there is no worked example, but worth noting in context.

**Suggested edits:**
- Add the two missing patterns to the "Common MBT Reasoning Patterns" list.
- Add a forward reference: "This module also covers governed domain analysis (Lesson 12) and policy/rule/exception/purpose/effect reasoning (Lesson 13)."
- Add one mini-exercise after the Concept Snapshot using a simple 2-sentence stimulus.

### Lesson 17-2: Step-by-Step Guide: Must Be True (12 blocks)

**Role:** The complete 4-step method for MBT questions.

**Strengths:**
- The 4-step process block is clear and correctly sequenced.
- Conditional Diagramming table is accurate and covers the most common natural-language forms.
- Quantifier Tracking paragraph correctly distinguishes "most/most → overlap" from "some/some → no overlap guaranteed."
- Wrong-Answer Patterns breakdown covers the five most common traps with clear badge labels.
- Worked Method Snapshot (top-shelf books / hardcovers / pre-1990 / digital edition) is a clean illustration.

**Issues:**
- **The 4-step method is purely prephrase-first.** Step 3 says "State your predicted deduction before reading the choices" and Step 4 says "Test each choice." This is the right workflow for clear deductions, but hard MBT questions often resist a clean prephrase. The mastery course version adds answer-driven POE as an explicit fallback path — this is the most practically important guidance the website version omits.
- **No "Governed Domain First" concept.** The mastery course adds a section explaining that on hard MBT questions, the first question is "Who exactly is governed by this rule?" The website version does not address this at all.
- **Two wrong-answer patterns missing** from the website version: "Domain Shift" and "Purpose-to-Rule Jump" — both central to the advanced drills.
- **No question-card.** The Worked Method Snapshot is a paragraph analysis, not an interactive question-card the student can attempt.
- **No "try it first" prompt** — not applicable since there is no question-card, but the absence of any practice opportunity is notable for the method lesson.

**Suggested edits:**
- Add Step 3.5 or modify Step 4: "If no clean prephrase emerges from Step 3, shift to answer-driven POE — test each answer against the certainty standard rather than against a prediction."
- Add a "Governed Domain First" subsection between Quantifier Tracking and Wrong-Answer Patterns.
- Add "Domain Shift" and "Purpose-to-Rule Jump" to the Wrong-Answer Patterns breakdown.
- Add a question-card after the Worked Method Snapshot.

## Structural Issues

### 1. Missing "Try It First" Prompts (All Drills, Lessons 17-3 through 17-10)

Every drill lesson presents a question-card followed immediately by the full analysis. There is no structured pause, no callout, and no instruction telling the student to attempt the problem before reading the solution. This is the module's most pervasive gap and the highest-priority fix. The drills are technically strong — the only thing missing is the student's moment of active engagement.

**Recommendation:** Add a callout or paragraph before each "Full Analysis" heading in lessons 17-3 through 17-10 with language such as: "Take a few minutes to attempt this question on your own before reading the analysis below."

### 2. Broken Worked Example in Lesson 17-6

The hardest lesson in the module (17-6) is the only one with a structural defect: the "Hard Drill" section presents a scenario description and then jumps directly to a full 5-answer analysis without ever showing the student the question stem or the answer choices as a question-card. A student reading this lesson cannot engage with the worked example at all — they can only read the instructor's analysis of a question they never saw.

**Recommendation (HIGH PRIORITY):** Reconstruct the question-card for lesson 17-6. The analysis refers to "(A) Dorsey's houses are both on the town council's approved list" through "(E) Dorsey has been a member of the town council longer than most other members." These five choices plus the question stem need to be added as a proper question-card block before the Full Analysis.

### 3. Placeholder Formatting Artifacts

Lessons 17-4 through 17-10 each contain `<br/><br/><br/>` placeholder blocks in a `paragraph` element. These were likely added to create vertical space between the concept section and the question-card but serve no pedagogical purpose. They add visual clutter.

**Recommendation:** Remove all `<br/>` placeholder blocks.

### 4. Lessons 17-12 and 17-13 Are Not Present in the Website

The mastery course has two additional lessons (Lesson12_GovernedSetsAndDomains.tsx and Lesson13_PolicyRuleExceptionPurposeEffect.tsx) that do not have corresponding website files. The website module ends at 17-11. The mastery course patterns introduced in 17-1's full version are therefore not drilled in the website content. This is a deliberate scope difference but worth noting for completeness.
